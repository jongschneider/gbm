### Validation Checklist:
**Navigation & Footer (US-006a):**

- [ ] Footer hides 'p: push' when tracked worktree selected
- [ ] Up/down navigation works, wraps at boundaries
- [ ] q/esc quits, space/enter outputs path and quits

**Pull/Push Operations (US-006b):**
- [ ] 'l' shows "Pulling..." then "Successfully pulled" message
- [ ] 'p' on ad-hoc shows "Pushing..." then "Successfully pushed" message

- [ ] Messages auto-dismiss after 3-5 seconds
- [ ] Git status column updates after operations

**Delete Confirmation (US-006c):**
- [ ] 'd' shows "Delete worktree X? (y/n)" prompt
- [ ] 'n' or 'esc' cancels and clears prompt
- [ ] 'y' deletes, shows success, removes row from table
- [ ] Success message auto-dismisses after 3-5 seconds

## 2026-01-10 - US-004
- Implemented AsyncRow type in pkg/tui/table.go for per-cell async loading
- AsyncRow holds map[int]string staticCells and map[int]*async.Cell[string] asyncCells
- NewAsyncRow(cells...string) creates row with initial static cells indexed by position
- WithAsyncCell(colIdx, cell) adds async cell and removes from static cells
- GetCell(colIdx) returns async cell View() or static string value
- ToTableRow(numCols) converts AsyncRow to table.Row for rendering
- Tick() calls Tick() on all async cells to update spinner animations
- IsLoading() checks if any async cell is still loading
- Updated Cell[T] to use github.com/charmbracelet/bubbles/spinner instead of manual spinnerFrames
- Cell.Tick() now updates spinner.Model with spinner.Tick() message
- Table.SetAsyncCell(rowIdx, colIdx, cell) adds async cell to specific location
- Table.Update() ticks all async rows and refreshes table display each frame
- Code builds, spinners animate correctly
- **Learnings for future iterations:**
  - Use bubbles/spinner.Line for standard spinner animation
  - spinner.Update(spinner.Tick()) advances animation frame
  - AsyncRow tracks spinners per-row and refreshes table on each tick
  - Table.updateTableRow() maintains consistency between AsyncRow and table.Row display
